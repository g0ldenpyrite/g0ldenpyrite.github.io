About

What is this project About ?

What is this project?

Our project is designed to predict the price of a laptop based on their different features such as size, GPU, CPU, memory space, operating system, and company. This product will ultimately make buying a laptop a quick and easy process that will fit the needs of the consumer.

What is the ultimate goal?

The ultimate goal of this product is to predict laptop prices so that the consumer that uses it can find the best match based on their preferences.

What is our data set?

Our data set listed out different computer products from various companies based on the features such as CPU and Screen Length.

Classification or Regression?

Our project is a regression model because it is predicting laptop price, which is a continuous value.

What is our MVP?

A website that takes user preferences for laptops. The website will then use those variables to find laptop prices that are within a certain price range.

Why does this matter?

If someone is planning to buy a laptop in the near future, this can be a simple and useful tool to get an estimated price for the laptop.

Data Cleansing

Why Data Cleansing?

Clean data increases productivity, accuracy and ensures quality results.

Removing Columns

Sometimes, dataframes can have columns that have little to no use for the final product that we are making. This is why we need to remove these unwanted columns.

Dealing with Null Values

Removing data that has no value, aka Null values. A Null represents an unknown or a missing value that affects the quality of the operations of processing, classification, or any other data analysis.

Encoding Data

Our data contained both numerical and categorical data, but the machine learning model used was unable to comprehend categorical data. This made it necessary for us to encode our data, transforming categorical data into numerical zeros and ones, allowing the machine learning model to understand the data.

Exploratory Data Analysis

Why EDA is required ?

Exploratory data analysis (EDA) involves using statistics and visualizations to analyze and identify trends in data sets. Exploratory data analysis is essential for any business. It allows data scientists to analyze the data before coming to any assumption. It ensures that the results produced are valid and applicable to business outcomes and goals. It helps us identify errors, outliers, and relationships in the data.

Machine Leaning Pipeline

What is Machine Learning Pipeline ?

Machine Learning Pipeline is a process of preparing data in order for it to be interpreted in a machine learning model. This process includes several steps such as pre-processing, cleaning, model implementation, and finally predictions

Machine Learning Pipeline

Data Cleansing / Feature Engineering

In order to clean our data-set, we have taken several steps including dropping unnecessary columns, removing null data and encoding the data.

Train : Test Split

In our supervised learning model, we want to split what data we have into separate train and test groups. The way our team decided to work with this type of algorithm was in order to simulate the most accurate prediction because of the complexity the information in a data turned out becoming; henceforth, splitting 80% of our data into the train group and 20% into the test group.

Imputation / Encoding

Instead of deleting whole columns just because of null values which would result in large loss of data, we used imputation and encoding to deal with the missing data. For example, we used the average of the columns to replace null values in instances of missing numerical data. Furthermore, we used the most frequently occurring value to replace missing categorical data.

Model Building

The particular models we want to display our conducted machine learning algorithms are Linear Regression, Random Forest and XGBoost.

Fine Tuning : GridSearchCV

Finally, we fine tuned our data through the GridSearchCV function which searches through a given set of parameter values and selects whichever combination gives the most correlation. This allowed us to get the best possible hyperparameter values in each of the machine learning models we incorporated.



Linear Regression

Linear Regression is a type of machine learning model that uses the correlation of an x variable (input) and tries to determine a y (output). Using given data points, linear regression creates a straight line of best fit to predict a trend on how the data will continue in future data points. Through training, linear Regression decreases the distance between the actual data points and the predictions, to as small as possible

RandomForest Regression

Random Forest uses a technique called bagging. This method uses a multitude of decision trees to release a prediction for the output. Random Forest models contain several working parts that make the way this model computes data uniquely. In Decision Trees, they start with the root of the tree/test sample input and follow branch resembling splits based on variable outcomes until a leaf node is reached and a results is given. In Ensemble Learning, multiple models are trained over the same test sample input data to which the the results are averaged between each model to finding a the best predictive result. The final method is Bootstrapping, this algorithm processes sampled subsets in our data-set randomly working with GridSearchCV to obtain results based upon preset parameters.

XGBoost Regression

A loss function in its optimization and use will be dependent on the type of problem being solved. Decision trees are used as the weak learner in gradient boosting process. An additive model adds weak learners to minimize the loss function allowing number trees to be added one at a time and existing trees in the model are not changed. Ensembles are constructed from decision tree models. Trees are added one at a time to the ensemble and fit to correct the prediction errors made by prior models in a type of ensemble machine learning model referred to as boosting.
